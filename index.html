PreGo UI Renewal Proposal

「出会い系の顔」を卒業し、
クラブハウスの品格をアプリに。

現状のUIは機能が揃っているのに、色が4系統（緑・ピンク・青・黄）に割れ、世界観が定まっていません。 本提案はブランドを「フェアウェイグリーン × アイボリー × 真鍮」の1系統に統一し、 マッチングを増やす3つのUX原則とともに再設計したものです。

提案日2026.07.04 対象prego-app.golf（Vite + React PWA） 作成森（株式会社エフオーピーシー）
01

デザイン原則 — 見た目の前に、構造を直す

1

色は1系統に絞る

主役はフェアウェイグリーン、格を出す差し色は真鍮ゴールドのみ。ピンク・青・黄のCTA混在を廃止し、「押すべきボタンは常に緑」を全画面で徹底します。

2

過疎を見せない

「7/14（火）2人」という供給の少なさを数字で晒す表示を廃止。日付起点のティーシート形式と公式コンペ枠で、常に画面が埋まっている状態を設計でつくります。

3

ゴルフ情報ファースト

プロフィールの主役を写真の雰囲気からベスト・平均・ラウンド回数のスコアカードへ。出会い系の空気を薄め、女性の登録ハードルと男性の真剣度の両方を改善します。

4

マッチの成果を資産に

ラウンド後にシェアしたくなるスコアカード画像を自動生成。UGC・紹介が「お願いして書いてもらうもの」ではなく「自慢したくて勝手に出るもの」になる導線です。

SCREEN 5 / 惜しいマッチ（新設）

「0件です」と言わない。
あと一歩を、橋渡しする

全国展開×少人数の宿命で、「日程は合うが遠い」「近いが日程が合わない」がマッチ不成立の大半を占めます。現状は「まだマッチ候補のお相手がいません」で終了。ここを「惜しい候補＋解決アクション」に変えます。

  • 日程◎×距離△ → 双方の拠点から等距離の中間地点コースを自動提案（拠点設定・市区町村精度は実装済みなので流用可能）
  • 距離◎×日程△ → 相手の空き日に合わせる日程リクエストを1タップ送信（Webプッシュ実装済み＝通知もすぐ届く）
  • プロフィールに移動許容度（〜60分/〜90分/遠征OK）を追加し、二値ではなくスコアでマッチ
  • それでも埋まらない日は公式コンペ枠が受け皿になる

05

実装メモ — エンジニア向け

導入はCSS変数の差し替えから

全面改修は不要です。まず既存コンポーネントの配色をこのトークンに寄せるだけで統一感の8割は回収できます。優先順位：

  1. CSSトークン導入＋既存ボタン/バッジの色統一（ピンク・青の廃止）
  2. 日程マッチ画面のティーシート化（人数表示の廃止）
  3. プロフィールのスコアカード先頭化・CTA一本化
  4. ラウンド録（新機能・シェア画像はcanvasかsatoriで生成）
:root {
  --fairway: #0C4633;   /* primary / CTA */
  --turf:    #1B7A4E;   /* secondary */
  --ivory:   #FAF8F2;   /* background */
  --brass:   #B08A3C;   /* rating / premium */
  --ink:     #20281F;   /* text */
  --radius:  14px;
  --font-brand: 'Marcellus', serif;
  --font-ui: 'Zen Kaku Gothic New', sans-serif;
}

デザインと同時に直したい仕様

  1. オファー課金の内包化 — 都度5,500円/22,000円をやめ、サブスク上位プラン（例：プレミアム月9,800円＝オファー月3回込み）へ。都度課金は「送らない理由」を作っている
  2. 公式コンペ機能 — 当面は運営が手動でイベント作成できる管理画面だけでよい（マッチング自動化は不要）
  3. 女性→女性の招待コード — 招待成立でコイン付与。ボトルネックは女性登録数
  4. 同性マッチ解禁 — 「仲間探し」への看板拡張。女性は同性でも無料のまま
  5. 惜しいマッチ — 中間地点提案は実装済みの拠点設定（市区町村精度）を流用。日程リクエスト通知は実装済みのWebプッシュに載せる
  6. フォトフレーム生成 — canvasかsatoriでサーバー/クライアント生成。投稿先は実装済みのフィード＋SNS

公開バンドルの調査で確認済みの既存資産：Supabase・Stripe・Webプッシュ・拠点設定（GPS/郵便番号）・AIマッチング提案・フィード投稿・記事・コイン/出金。上記の新機能はいずれも既存部品の組み合わせで実装可能です。

この提案書自体がHTML/CSSのみで書かれているため、各画面のマークアップはそのまま実装の参考にできます。
