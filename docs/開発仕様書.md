# PreGo リニューアル 開発仕様書

**版:** v26準拠（2026-07-18）
**動作デモ:** https://shuhei-mori.github.io/prego-design-proposal/app/ （実装の正はこのデモ。本書と食い違う場合はデモの挙動を優先し確認すること）
**デモの操作:** 画面左下ピル＝「切替」(男性SHU⇔女性みどり)／「配色」(B/E/G/1巡回)／「↺」(状態リセット)。URLパラメータ `?theme=b|e|g|1` で直接指定可。
**「1」について:** 配色パターンではなく**新機能フラグ**（配色はGを継承）。本書で【機能1】と記した仕様は `?theme=1` でのみ有効＝リニューアルで新規開発する部分。無印は現行アプリの再現＋全体改善。

## 1. デザインシステム

### 1.1 カラートークン（CSS変数）

デモの `style.css` の `:root` と `body.theme-b / theme-e / theme-g` が正。主要値：

| 変数 | 役割 | B ミント | E エメラルド | G ミックス（推し） |
|---|---|---|---|---|
| --fairway | プライマリ（ボタン・リンク・アイコン） | #1CA57F | #007658 | #1CA57F |
| --fairway-deep | プライマリ濃 | #0E6B54 | #00553F | #0E6B54 |
| --turf / --turf-soft | セカンダリ／淡背景 | #2FBD97 / #DFF6EC | #009395 / #DCF0EC | Bと同じ |
| --ivory | 画面背景 | #F1FBF6 | #F0F8F5 | #F1FBF6 |
| --paper | カード面 | #FFFFFF | #FFFFFF | #FFFFFF |
| --ink / --ink-soft | 本文／補助文字 | #1E3A31 / #628579 | #0F2B24 / #5B7A72 | Bと同じ |
| --brass / --brass-ink / --brass-soft | アクセント金（バッジ・認定） | #27A986系 | #009395系 | Bと同じ |
| --line | ヘアライン境界 | #D5EEE3 | #D2E7E1 | Bと同じ |
| --danger | 危険（退会等） | #B4543A | 同左 | 同左 |

**Gの構成ルール:** 基本は**B**の全変数。**ログイン画面系**（--grad-login, --login-*, --glass-*, --chipg-*, --tbtn-*）と**マイページヘッダー**（--myhead-bg: linear-gradient(180deg,#007658,#00654E)・ヘッダー内のみ --on-brand:#FFF）だけ**E**を注入。
**金色の面積は5%以下**（バッジ・ライン・アイコンのみ。ベタ面に使わない）。
いいねのアクティブ色: #FF7A93（ハート塗り）。

### 1.2 タイポグラフィ

| 用途 | フォント | 備考 |
|---|---|---|
| 本文UI | 'Zen Kaku Gothic New' (400/500/700/900) | --font-ui |
| 英字ブランド見出し・キャプション | 'Marcellus' | --font-brand。PREMIUM GOLF等。**数字には使わない** |
| **数字全般**（金額・pt・スコア・日付・カード番号・SMS） | **'Outfit' (300/400/500)** | --font-num。大きい数字（残高40px・プラン価格26px・スコア）はweight 300 |

Google Fonts: `family=Marcellus&family=Outfit:wght@300;400;500&family=Zen+Kaku+Gothic+New:wght@400;500;700;900`

### 1.3 レイアウト・トークン

- **余白:** 8の倍数のみ（8/16/24/32px）。カード内padding 16px、セクション間24px、画面左右18px
- **角丸:** 3段階のみ — チップ/ピル 99px、カード・入力 16px（小ボタン12px）、シート/モーダル 20px
- **影:** 2種のみ — カード `--sh-card: 0 6px 16px rgba(32,40,31,.08)`／浮遊物(シート・ピル) `--sh-float: 0 12px 28px rgba(32,40,31,.18)`。境界は影でなく1pxヘアライン(--line)
- **画面幅:** モバイル最大430px（#stage）。タブバー高 --tab-h:76px

### 1.4 モーション

- **イージング1種:** `--ease: cubic-bezier(.2,.8,.3,1)`
- **duration 3種:** タップ150ms／画面遷移250ms／シート400ms
- **ページ入場:** pageIn 340ms（opacity 0→1・translateY 12px→0）。**同一画面内の状態更新（いいね等）では再生しない**（デモは body.no-page-anim で抑止。本番はdiff更新なら自然に不要）
- **ハート:** タップで scale 1→1.3→1（350ms）＋アウトライン→塗り(#FF7A93)
- **ロゴ:** ログインで「PREMIUM GOLF」→公式ロゴ「PreGo」へモーフ（logoIn 800ms・delay 2.35s）。ロゴ画像は白1色PNG(1871×461)をCSSマスクで使い、テーマ色を background で着色する方式（テーマごとに画像不要）
- **祝福演出 celebrate():** マッチ成立承諾・ラウンド確定・サブスク加入の3箇所のみで発火。中央チェックサークル84px（ckPop 500ms）＋紙吹雪18枚（8×12px・色 #D9B96A/#68A083/#7AE0BD/#FFF/#009395・放射状に90〜200px飛散・cfFly 1s・delay 12ms刻み）→1.2秒で自動消滅。**普段は静かに、特別な瞬間だけ祝う**
- **アイコン:** 全て細線SVG（stroke 1.4〜1.8、丸端）。♥★足あとはアウトライン、選択時のみ塗り

## 3. 機能仕様

### 3.1 女性のスタイル（仲間探し／おもてなし）

- 女性はプロフィールで**スタイルを自己選択**: `st:'n'`(仲間探し=謝礼不要・割り勘前提) / `st:'o'`(おもてなし=謝礼オファー受付)
- 表示: 一覧カード・プロフィールにバッジ（仲間探し=白チップ／おもてなし=金チップ）
- **誘い制御（重要）:** 仲間探し女性→謝礼オファー不可・「ラウンドに誘う（謝礼不要）」のみ／おもてなし女性→謝礼オファーのみ。タダ誘いがおもてなし女性に届かない構造
- ホームのフィルタチップ「仲間探し（謝礼不要）」「おもてなし」で絞り込み

### 3.2 【機能1】認定／一般ゴルファー制度（女性）

- 登録時は全員**一般**。以下3条件を満たすと**認定**：
  1. アプリ上でマッチし、ラウンド1回以上（**打ちっぱなし・シミュゴルフを含む**）
  2. 双方でレビュー済み、かつ**相手から受けた評価が★4以上**
  3. プロフィール充実が運営設定の基準以上（デモ基準: **写真3枚以上・自己紹介200字以上・選択項目全入力**）
- 表示: 女性カードに「認定」(金)／「一般」(白)バッジ。女性マイページに**条件チェックリスト**（達成✓/未達グレー・「認定まであとN条件」チップ・未達項目への導線ボタン）
- 認定の特典: 謝礼単価が上（ランク制と連動）・コンペ開催権（3.9）

### 3.3 誘い導線（仲間探し女性向け・サブスクのみで誘える）

シート構成（上から）:
1. 【機能1】**会う形式** 縦積みカード3択: ラウンド(ゴルフ場で18ホール・1日)／シミュゴルフ(1〜2時間・駅近・雨でもOK)／打ちっぱなし(1時間・手ぶらOK・気軽)。シミュ・打ちっぱには「はじめまして向き」タグ。カード=左アイコンタイル38px・タイトル13.5px・説明10.5px・右に丸チェック22px。選択中: 枠--fairway・タイル塗り
2. **日程**（お互いの空き日をハイライト）
3. **プレー代の宣言(必須)** 3択: 「★プレー代はこちらで持ちます（返信率が上がります）」推奨／割り勘／相談。形式が練習系のときは文言「費用」に変化
4. 注記「宣言した内容はお相手に表示され、あとから変更できません」
5. 送信→チャットへ遷移し、sysメッセージ「⛳ {日付} の{形式}にお誘い（{宣言}）」＋本人メッセージが入る
- 未加入男性が開くとペイウォール表示

### 3.4 オファーフロー（おもてなし女性向け・謝礼つき）

- 【機能1】**会う形式3択**をフロー先頭に追加: ラウンド=ランク料金／シミュ・打ちっぱ=**女性が設定した謝礼額**を表示（「謝礼 ¥8,800・本人設定」）。女性が受付OFFの形式は**「受付停止中」で選択不可**
- **価格計算式:** ラウンド=ランク価格表（下表）。練習系=`謝礼額 ÷ 0.8 を100円単位に丸め`（配分80/20は全形式で共通）
- 合流方法: **現地集合（推奨・各自でコースへ・バッグは宅急便OK）／車送迎（認証ドライバーのみ）** の2択。**駅集合は廃止**
- 場所: ラウンド=ゴルフ場選択（中間地点から候補）。練習系=「中間エリアの提携施設から候補を提案・確定時に3候補から選択」
- 価格内訳は常時表示: 料金／謝礼80%／サービス利用料20%／キャンセル規定
- ランク価格表（女性受取=80%）: WHITE ¥5,500(受取4,400)→BRONZE→SILVER→GOLD→BLACK ¥27,500(受取22,000)

### 3.5 【機能1】お誘い設定（女性）

- マイページ→「お誘い設定」。シミュゴルフ／打ちっぱなしそれぞれに:
  - **受ける/受けないトグル**（OFF=男性のオファー画面で受付停止中）
  - **謝礼額の指定**: ¥3,300／5,500／8,800／11,000／16,500 から選択（受取は表示額の100%＝価格転嫁は男性側料金で行う）
- ラウンドの謝礼はランク制のため本人変更不可（注記表示）

### 3.6 【機能1】サブスク（チケット制・¥980廃止）

| プラン | 価格 | 内容 |
|---|---|---|
| スタンダード | ¥2,480/月 | メッセージ＋ラウンド誘い（プレー代宣言）使い放題 |
| プレミアム(推奨) | ¥9,800/月 | ＋謝礼オファーチケット**月1枚**（WHITE相当・**当月末失効・繰越なし**） |
| 年間プラン | ¥23,760/年 | スタンダード内容＋チケット**3枚**（月あたり¥1,980表記） |

- チケット1枚=デビューオファー1回（相手に謝礼¥4,400）。上位ランク指名は差額のみ追加課金
- 加入中画面に「オファーチケット 残り1枚（7/31失効）」行を表示
- 無料会員: 閲覧・いいね・受信返信まで。メッセージ送信/誘い/オファーでペイウォール

### 3.7 チャット

- **既読:** 自分のメッセージは、それ以降に相手のメッセージが存在する場合「既読」表示（メタ列: 既読＋時刻・9px・右寄せ）
- **メッセージ種別:** `me / them / sys / card`。cardは `kind:'match' | 'plan'`
- **マッチ成立カード（kind:match）:** オファー承諾・ラウンド確定の**時点でメッセージとして挿入**（時系列に並ぶ。画面下部固定ではない）。内容: ✓ヘッダ「{日付} {形式}マッチ成立/確定」・場所・謝礼（オファー時のみ「n コイン確定済み」）・安全機能行。金枠(--brass)・最大320px・中央寄せ。**段取り未提案のときのみ**「当日の段取りを提案する」ボタンを内蔵
- **段取り提案カード（kind:plan）:** 日程/集合時間/集合場所の表形式＋状態行。状態遷移: 「お相手の返事待ち」→相手OK→「✓ お相手がOKしました・スケジュール登録済み」（両者のスケジュール登録＋前日リマインドを発火）
- **段取り提案シート:** 集合時間は形式連動 — ラウンド=6:40/7:00/7:20/7:40/8:00/8:30/9:00・場所=クラブハウス前/練習場受付/コース玄関。練習系=10:00〜21:00の8択（夜対応・「仕事帰りの夜集合もOK」注記）・場所=施設受付前/駅改札（施設最寄）/施設ビル1F入口
- **マッチ成立時の遷移:** 承諾→祝福アニメ→**約1秒後に該当チャットへ自動遷移**
- 上部バー: **未確定の相手のみ**「ラウンド確定」CTA（確定するとレビュー・安全機能・実績カウント有効の説明つき）。確定済みはバー非表示（カードが担う）

### 3.8 日程マッチ（旧ティーシート）

- **改名:** 画面名・フッタータブとも「日程マッチ」
- **月切替:** ヘッダー右「‹ JULY 2026 ›」で月移動。日付ストリップ・候補・コンペが月に追従
- **希望日登録:** 画面上部にカード「あなたのプレー希望日」— 日付チップをタップで登録/解除。**登録0件の間は候補を表示せず**「登録するとお相手が表示されます」の空状態（現行アプリ準拠）
- **エリア登録:** 同カード内にエリアチップ＋「変更」→ 都県選択シート（複数可）
- 候補リスト: その日に空きがある相手（ランクリング・Best/Ave・エリア）＋公式/ホスト開催コンペ＋「惜しいマッチ」導線
- **ホームとの連動:** ホームの「日程が合う」チップ→日程マッチへ遷移。ホーム上部バナーはフィルタ連動 — すべて=「プレー希望日を登録するとマッチ率が3倍になります →」(タップで日程マッチへ)／仲間探し=「謝礼不要。プレー代の宣言だけで誘えるお相手です」／おもてなし=「認定ゴルファーに謝礼つきオファーで誘えます」

### 3.9 【機能1】コンペ

**運営開催:** 管理画面から作成（日時・会場・形式・男女枠・参加費・キャンセル規定）→公開→エントリー管理。表示ラベル「PREGO OPEN」

**女性ホスト開催（承認制）:**
- 開催権: **認定ゴルファーのみ**。マイページの「コンペを開催して報酬を得る」→申請フロー:
  1. 形式: ラウンド／シミュ練習会／打ちっぱなし会（形式で提携会場リストが切替）
  2. 日程・会場（**提携先からのみ**選択）
  3. **共同ホスト**トグル: 友人を招待リンクで誘える。**未登録の友人も招待リンクから登録して参加可**（女性→女性の獲得チャネル）
  4. 男性枠(2/4/6/8名)・参加費(¥5,000/8,000/10,000/15,000)
  5. **報酬計算（リアルタイム表示）:** 幹事報酬=参加費合計×30%（100円丸め）。共同ホスト時は2等分。例: 4名×¥8,000×30%÷2=**¥4,800/人**
  6. 申請→運営審査→公開。会場費・保険・キャンセル規定は運営テンプレ・当日運営スタッフ1名同行
- 一覧表示: ホスト開催はラベル「HOSTED BY 認定ゴルファー」＋ホスト2名のアバター

### 3.10 プロフィール・登録まわり

- **居住地域（非公開・必須化推奨）:** プロフィール編集に「お住まいの地域」= 都道府県select＋市区町村テキスト。注記「お相手には表示されません。距離が近い方とのマッチングにのみ使用します」（シールドアイコン付き）。距離マッチングのソートに使用
- 写真は複数枚（スワイプ・ドット・n/n表示）。本人確認バッジ・実名非公開
- ラウンド録: OUT/IN入力→**TOTAL自動計算**（手入力欄なし・表示順 OUT→IN→TOTAL）。canvasでスコアカード画像生成（右下に公式ロゴ・PNG保存）

## 4. データモデル（デモ→本番読み替え）

デモは localStorage キー `prego-demo` に単一オブジェクトS。本番はAPI/DBに読み替え。

```
S = {
  role, theme, subActive, verified,
  points(男性pt), coins(女性コイン),
  likes{}, favs{},
  myDates:['7/14',...],    // プレー希望日（日程マッチ登録）
  myArea:['千葉','東京'],   // プレーエリア
  homeArea:'千葉県 市川市', // 居住地域（非公開・距離マッチ用）
  fset:{ sim:bool, simFee:number, range:bool, rangeFee:number }, // 女性お誘い設定
  sentOffers:[{to,date,meet,course,mode,price,status}],
  recvOffers:[{from,date,meet,course,mode,reward,status}],
  chats:[{id, msgs:[{who:'me'|'them'|'sys'|'card', kind?, t?, tm?, ...}]}],
  fixed:{ [相手id]: {date,course,mode,offer,reward} },  // マッチ確定
}
女性ユーザー: { st:'n'|'o', cert:bool, sim:{ok,fee}, rng:{ok,fee}, tier, rating, dates[], area[], ... }
```

※デモ限定の擬装（本番不要）: S.bridge/S.bridgeM（男女デモ間の反映）、↺リセット、デモ自動返信、配色切替UI。
